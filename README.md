# AbaChat（阿巴聊天）

基于 WebSocket 的私密聊天室应用，支持一对一和多人聊天模式，通过房间口令实现访问控制。

## 技术架构

- **后端**: Node.js + WebSocket (ws) + HTTP 静态服务器
- **前端**: 原生 HTML/CSS/JavaScript，无框架依赖
- **通信**: WebSocket 实时双向通信
- **依赖**: `ws` (WebSocket 服务器)、`uuid` (用户 ID 生成)

## 功能清单

### 房间系统

| 功能 | 描述 |
|------|------|
| 房间创建 | 通过输入房间口令自动创建房间 |
| 访问控制 | 房间口令验证，保护隐私 |
| 聊天模式 | 支持一对一（最多2人）和多人聊天（最多10人）两种模式 |
| 空房间清理 | 房间无人时自动删除 |

### 用户管理

| 功能 | 描述 |
|------|------|
| 昵称设置 | 可选昵称，留空则自动生成 |
| 昵称去重 | 重复昵称自动追加数字后缀（如 user_1） |
| 用户列表 | 实时显示房间内所有在线用户 |
| 加入/离开通知 | 用户进入或离开时显示系统消息 |

### 消息功能

| 功能 | 描述 |
|------|------|
| 文本消息 | 支持最长 2000 字符的文本消息 |
| 消息时间戳 | 每条消息显示发送时间 |
| 消息类型 | 支持 text、emoji、image 三种消息类型 |
| 自动表情识别 | 单独发送单个 emoji 时自动识别类型 |
| XSS 防护 | 消息内容经过 HTML 转义处理 |
| 已读回执 | 可实时查看消息被阅读状态（已阅读） |

### 实时交互

| 功能 | 描述 |
|------|------|
| 输入指示器 | 显示"XXX 正在输入..."提示，3秒后自动消失 |
| 在线状态 | 头部显示当前连接状态 |
| 心跳检测 | 30秒间隔保活检测，10秒超时断开 |

### 界面功能

| 功能 | 描述 |
|------|------|
| 登录界面 | 模式选择、口令输入、昵称输入 |
| 等待界面 | 单人等待时显示等待动画和人数统计 |
| 聊天室界面 | 消息显示区、输入区、用户列表区 |
| 房间已满提示 | 房间满员时显示友好提示 |
| 表情面板 | 72个常用 emoji 表情，点击即发送 |
| 用户面板 | 可折叠的在线用户列表 |
| 响应式设计 | 适配移动端和桌面端 |

### 浏览器适配

| 功能 | 描述 |
|------|------|
| 页面关闭提醒 | 离开页面时自动发送离开消息 |
| 回车发送 | 支持 Enter 键发送消息 |
| 表情点击发送 | 点击表情直接发送 |
| 自动重连 | 连接断开后提示用户 |

## 消息协议

### 客户端 -> 服务器

| 消息类型 | 用途 |
|----------|------|
| `join` | 加入房间 |
| `chat_message` | 发送聊天消息 |
| `typing` | 发送输入中状态 |
| `stop_typing` | 停止输入状态 |
| `read_receipt` | 发送已读回执 |
| `leave` | 离开房间 |
| `ping` | 心跳检测 |

### 服务器 -> 客户端

| 消息类型 | 用途 |
|----------|------|
| `join_success` | 加入成功 |
| `join_error` | 加入失败 |
| `user_joined` | 用户加入通知 |
| `user_left` | 用户离开通知 |
| `chat_message` | 收到聊天消息 |
| `typing` / `stop_typing` | 输入状态变化 |
| `read_receipt` | 已读回执 |
| `pong` | 心跳响应 |

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| PORT | 3000 | 服务器端口 |

## 使用说明

### 系统要求

- Windows 操作系统
- 已安装 Node.js（若未安装，启动时会提示下载地址）
- 网络连接（首次运行需要下载依赖）

### 启动方法

**方法一：双击启动（推荐）**

1. 找到并双击 `start.bat` 文件（或 `start.ps1`）
2. 若弹出安全提示，点击"运行"
3. 等待脚本自动完成安装和启动

**方法二：使用命令行**

```bash
# 进入项目文件夹
cd AbaChat

# 安装依赖
npm install

# 启动服务器
npm start

# 或使用开发模式（自动监听文件变化）
npm run dev
```

**方法三：使用 PowerShell**

```powershell
powershell -ExecutionPolicy Bypass -File start.ps1
```

### 使用步骤

1. 启动成功后，打开浏览器访问：http://localhost:3000

2. 选择聊天模式：
   - **一对一**：适合两人私密聊天
   - **多人聊天**：适合小组讨论（最多10人）

3. 输入房间口令：
   - 与你想聊天的人使用相同的口令
   - 口令可以是任意字母或数字组合
   - 例如：abc123、love、meeting 等

4. 设置昵称（可选）：
   - 可以输入自定义昵称
   - 留空将自动分配随机昵称

5. 点击"进入聊天室"，等待对方加入即可开始聊天

### 功能说明

- 实时文字聊天：发送和接收即时消息
- 表情支持：点击笑脸按钮选择表情
- 在线用户：查看当前房间内的在线人数和用户列表
- 输入提示：显示对方正在输入的状态
- 已读回执：查看消息是否被对方阅读
- 离开聊天室：随时可以退出房间

### 注意事项

1. 确保所有参与者都使用相同的房间口令
2. 关闭启动窗口将停止服务器，所有连接会断开
3. 建议使用现代浏览器（Chrome、Edge、Firefox 等）
4. 聊天记录仅保存在当前会话中，关闭页面后不会保留
5. 请勿在聊天中发送敏感信息或个人隐私

### 常见问题

**Q: 启动时提示"未找到 Node.js"**

A: 请从 https://nodejs.org/ 下载并安装 Node.js

**Q: 无法进入聊天室**

A: 请确认：
- 所有参与者使用相同的口令
- 选择了相同的聊天模式（一对一或多人）
- 房间人数未达到上限

**Q: 聊天消息发送不出去**

A: 请检查网络连接，或尝试刷新页面重新进入

## 优化清单

### 一、安全问题

#### 通信安全

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 缺乏 HTTPS/WSS | 高 | 当前使用明文 WebSocket，生产环境存在安全风险 |
| 房间口令明文传输 | 高 | 口令在网络中明文传输，建议使用 TLS 加密 |

#### 输入验证

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 房间口令无验证 | 中 | 未验证口令格式和长度，可能导致正则注入 |
| 昵称无过滤 | 中 | 昵称可能包含特殊字符，建议限制长度和格式 |
| 消息内容验证不完善 | 低 | 仅验证长度，未检查其他恶意内容 |

### 二、稳定性问题

#### 连接管理

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 缺乏断线重连机制 | 高 | WebSocket 断开后无自动重连，用户需手动刷新 |
| 心跳超时未生效 | 中 | `HEARTBEAT_TIMEOUT` 常量定义但未实际使用，超时逻辑不完整 |
| 消息发送无确认 | 中 | 消息发送后无服务器回执，可能丢失 |

#### 错误处理

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 全局错误捕获缺失 | 中 | 消息解析错误仅打印日志，未通知客户端 |
| 房间满员后无友好提示 | 低 | join_error 返回后可优化用户体验 |

### 三、功能缺失

#### 消息功能

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 消息无持久化 | 高 | 服务器重启后聊天记录全部丢失 |
| 无法编辑/删除消息 | 中 | 发送错误消息后无法撤回或修改 |
| 无消息搜索 | 低 | 历史消息无法搜索 |
| 无图片上传 | 中 | 仅支持图片 URL，不支持本地上传 |

#### 通知功能

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 无声音提示 | 低 | 新消息无声音通知 |
| 无桌面通知 | 低 | 页面后台时无系统通知 |
| 无未读消息标记 | 低 | 无法区分已读/未读消息 |

#### 管理功能

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 无管理员权限 | 中 | 无法踢出用户或禁言 |
| 无房间设置 | 低 | 无法修改房间名称或密码 |
| 无用户黑名单 | 低 | 无法屏蔽特定用户 |

### 四、用户体验

#### 界面交互

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 无深色模式 | 低 | 无主题切换功能 |
| 输入指示器冗余 | 低 | 客户端和服务器同时维护 typing 状态，可能冲突 |
| 表情面板定位问题 | 低 | 在小屏幕上可能超出视口 |

#### 兼容性

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| Emoji 正则兼容性 | 中 | `/^\p{Emoji}/u` 在旧版浏览器可能不支持 |
| 移动端体验待优化 | 低 | 部分交互在移动端不够顺畅 |

### 五、性能问题

#### 服务器端

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 房间数据无内存限制 | 中 | 大量房间可能占用过多内存 |
| 无连接数限制 | 中 | 单个 IP 可创建大量连接 |

#### 客户端

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 消息列表无虚拟滚动 | 低 | 大量消息时可能影响性能 |
| 无消息分页加载 | 低 | 进入房间时加载所有历史消息（如有） |

### 六、代码质量

#### 代码结构

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 前后端未分离 | 低 | 可考虑拆分服务架构 |
| 状态管理混乱 | 低 | 全局变量过多，可使用模块化设计 |

#### 日志与监控

| 问题 | 严重程度 | 描述 |
|------|----------|------|
| 日志过于简单 | 中 | 仅 console.log，无结构化日志 |
| 无监控告警 | 中 | 无连接数、消息量等指标监控 |
| 无请求追踪 | 低 | 难以排查问题根因 |

### 优化优先级

#### 高优先级（建议立即修复）

1. WebSocket 断线重连机制
2. 心跳超时逻辑完善
3. HTTPS/WSS 支持
4. 消息持久化（可选）

#### 中优先级（建议近期修复）

1. 房间口令格式验证
2. 昵称过滤
3. 管理员功能（踢人、禁言）
4. 图片上传功能

#### 低优先级（可延后处理）

1. 深色模式
2. 声音/桌面通知
3. 消息搜索
4. 历史消息分页
